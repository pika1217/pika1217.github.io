Git常用操作

发表于 2017-09-08 | 分类于 工具
  1. git add 将文件添加到仓库

  2. git commit -m “” 将文件提交到仓库

    //每次git commit会提交所有添加到仓库的文件，所以要分为两步，add和commit，这样可以分开提交并且添加不同的提交说明

  3. git status 查看当前仓库状态
  4. git diff 查看具体修改内容
  5. git log 显示从最近到最远的提交日志、git log –pretty=oneline 信息概要、回退版本之后最新的记录会删除
  6. git reflog 显示每一次命令
  7. git reset –hard HEAD~num //num代表需要回退的版本个数 1代表前一个，这样就能从新往旧回退了
  8. git reset –hard hash //hash代表某次提交的hash值，这样就能从旧往新回退了，也能使用任何一次的hash跳转到任何一个版本
  9. 工作区和暂存区

    文件夹算是一个工作区，隐藏的目录文件.git是Git的版本库，其中有一个stage的暂存区，以及一个Git自动创建的master，以及一个执行master的指针HEAD

  10. Git跟踪并管理的是修改，而非文件。只有add进去暂存区才会被commit。
  11. git checkout – fileName 回到最近一次的git add或者git commit的状态
  12. git reset HEAD fileName 将暂存区的文件恢复到工作区
  13. 删除文件
    1. 工作区rm删除之后，Git会提示是继续删除仓库中的还是恢复之前的文件
    2. git rm fileName 仓库中的文件也删除
    3. git checkout fileName 恢复工作区的文件

远程仓库

分布式版本系统的最大好处之一是在本地工作完全不需要考虑远程库的存在，也就是有没有联网都可以正常工作，把内容都存放在本地仓库中，而SVN在没有联网的时候是拒绝干活的！当有网络的时候，再把本地提交推送一下就完成了同步，真是太方便了！

  1. 关联一个远程库 git remote add origin git@github.com:pika1217/gitTest.git
  2. git push -u origin master 第一次推送master分支的所有内容
  3. git clone 克隆远程仓库
  4. git branch dev 创建一个dev分支
  5. git checkout dev切换到这个分支，两步可以直接写出git checkout -b dev
  6. git branch 查看当前分支
  7. git merge dev 将指定分支合并到当前分支
  8. git branch -d dev 删除分支
  9. 开发一个新feature，最好新建一个分支；如果要丢弃一个没有被合并过的分支，可以通过git branch -D 强行删除。

    git fetch和pull的区别

Git中从远程的分支获取最新的版本到本地有这样2个命令：

  1. git fetch：相当于是从远程获取最新版本到本地，不会自动merge

以上命令的含义：

首先从远程的origin的master主分支下载最新的版本到origin/master分支上；然后比较本地的master分支和origin/master分支的差别；最后进行合并。上述过程其实可以用以下更清晰的方式来进行：

从远程获取最新的版本到本地的tmp分支上之后再进行比较合并

  1. git pull：相当于是从远程获取最新版本并merge到本地

上述命令其实相当于git fetch 和 git merge 在实际使用中，git fetch更安全一些。因为在merge前，我们可以查看更新情况，然后再决定是否合并结束。

多人协作的工作模式通常是这样：

首先，可以试图用git push origin branch-name推送自己的修改；

如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；

如果合并有冲突，则解决冲突，并在本地提交；

没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功！

如果git pull提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，用命令git branch –set-upstream branch-name origin/branch-name。
